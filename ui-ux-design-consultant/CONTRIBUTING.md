# Contributing to UI/UX Design Consultant

Thank you for your interest in contributing to this Claude Code skill!

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or request features
- Include clear reproduction steps for bugs
- For feature requests, explain the use case and expected behavior

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test your changes thoroughly
5. Commit with clear messages (`git commit -m "Add: description"`)
6. Push to your fork (`git push origin feature/your-feature`)
7. Open a Pull Request

### Code Guidelines

**For Python scripts:**
- Follow PEP 8 style guidelines
- Include docstrings for functions
- Output JSON for programmatic consumption
- Handle errors gracefully with informative messages
- Test with various project structures

**For reference files:**
- Use clear, actionable language
- Include code examples for every pattern
- Organize with clear headings and sections
- Keep framework-specific advice accurate and up-to-date

**For templates:**
- Use consistent `{{PLACEHOLDER}}` syntax
- Include all required sections
- Test with realistic data

### Areas for Contribution

We especially welcome:

- **Framework support:** Angular, Astro, Qwik, Solid
- **Component library guides:** MUI, Chakra, Ant Design, shadcn/ui
- **Audit scripts:** Animation patterns, bundle size impact, performance metrics
- **Accessibility:** Enhanced WCAG checks, screen reader testing patterns
- **Internationalization:** RTL layout considerations, locale-specific design patterns

### Testing

Before submitting:

1. Test audit scripts against real projects
2. Verify SKILL.md workflow executes correctly
3. Check that all file references resolve
4. Validate JSON output from scripts

## Code of Conduct

- Be respectful and constructive
- Focus on the work, not the person
- Welcome newcomers and help them contribute

## Questions?

Open an issue with the "question" label or reach out through GitHub Discussions.
